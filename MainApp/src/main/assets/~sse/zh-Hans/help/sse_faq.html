常见问题(FAQ) - 秘密空间加密器

我如何将数据迁移到新设备？

密码保险箱：
- 密码保险箱具有导出/导入功能（您可以选择经过压缩且完全加密的 .pwv 文件格式和未压缩可编辑的 .xml 格式）。
- 对于 Android 5.0 及更高版本 - 还可以使用（系统提供的）标准备份/还原功能。

手动传送：
1.将保险箱导出为 pwv 文件（使用 密码保险箱应用中的“更多”按钮） - 它将在应用的导入/导出目录
（默认为：[内部存储空间]/SSE/或[内部存储空间]/Android/media/com.paranoiaworks.unicus.android.sse/）中显示
2.将该文件复制到新设备上的导入/导出目录
3.导入该文件（启动密码保险箱 → 取消密码对话框 → 点击帮我登录！ → 选择导入保险箱 → 选择要导入的文件）
注意：您可以在应用的设置中检查/更改当前的导入/导出目录 - （设置 → 通用设置: 导入/导出路径）

文本 + 文件加密：
这些应用处理文件，因此，您可以将它作为任何其它文件进行备份/传送。文本加密 (text encryptor) 的默认目录为 [内部存储空间]/SSE/Texts/或[内部存储空间]/Android/media/com.paranoiaworks.unicus.android.sse/Texts/ 及 ../Steganograms/，文件加密 (file encryptor) 会在原始（未加密）文件所在的同一目录中保存一个新档案（这是默认设置）。可以在设置 → 文件加密: 加密文件目标路径中更改目标路径。

注意：卸载应用程序时，目录../Android/

PRO 版本许可证：我可以重复使用已购买的 SSE，还是需要重新购买？

此许可证终生有效，并且使用许可证的设备数量不受限制 — 购买的许可证与您的 Google (Google Play) 账户绑定在一起。使用许可证的设备数量将不受限制（只要您用购买许可证时使用的账户登录即可）。在初次启动时，应用应该会自动激活；如果无法激活，请前往工具箱 → 升级到 Pro 版并点击升级到 Pro 版版按钮。此时会显示一条警告消息（类似于“您已拥有此项目”），然后，应用将会被激活。此外，您还可以激活任意数量的桌面版本（更多信息）。如果您在此过程中遇到任何问题，请随时与我们联系。


我忘记了密码。您可以帮助我吗？

不，此应用没有后门。这正是我们开发此应用的目的。所有数据都使用密码进行了加密（从数学角度做出了改变）。如果没有密码，您将无法撤销此过程并“重建”原始数据。我们无法帮助您。没有人可以做到。获取正确密码是唯一的途径


此应用为什么不使用指纹来加密文件/文本以及访问密码保险箱？

所有数据都经过完全加密（从数学角度做出了改变），并且加密密钥必须完全匹配才能进行解密。如果没有100%正确的密钥，您将无法撤销此过程并“重建”原始数据。指纹通常用于身份验证 - 原始指纹存储在某个地方，并在身份验证过程中进行比较 - 但数据隐藏在这个过程之后。没有任何可靠的方法可以将指纹用于加密（如何通过指纹生成强大的加密密钥）- 到目前为止，在这方面只进行了少数几次不太可靠的实验。而且，指纹并不属于秘密，而且无法更改。如果某人拥有您指纹的副本（如果真正想要获取的话，任何人都可以相当轻松地获取 - 由于您几乎将密码写在您可以碰到的任何位置），那么，对他/她/他们来说，您的所有数据（过去、现在、将来）将唾手可得。S.S.E. 绝不会将生物特征数据用于加密。


为什么我不能使用其它采用相同加密算法的应用？例如，使用 AES 来解密文件/文本？

这不仅与加密算法有关，还涉及到其它许多因素：加密模式（CBC、CFB、PCBC、OFB、CRT、CWC、EAX...），密码处理（KDF - 密码推导函数），完整性检查，文件归档类型，压缩等。请单击此处了解详细的格式规范。

文件加密相关

我的文件在加密后仍然可见。是应用出现问题，还是我操作出错？

SSE 文件加密作为归档程序运行（将创建一个新的 .enc 文件）。您可以在加密过程完成后删除/擦除原始文件，也可以让系统自动执行此操作：设置: 文件加密 → 加密后擦除源文件


我的图片（或视频）已加密/删除，但我仍然可以在图片查看器/图库应用中看到缩略图。我该怎么办？

市面上存在许多图库/图片查看器应用，因此，这个问题并没有通用的解决办法。删除图片文件后，您的图库软件自身仍存有该文件的“缩略”副本（缩略图），是否更新此状态将取决于图库软件。缩略图可能存储（缓存）在图库软件的数据空间内（因此，没有其它应用有权访问该数据）。


Android 4.4 及更高版本：外置 SD 卡（或 U 盘）处于只读状态（或不可见）。消息：EACCESS（没有权限）。

Android 4.4 不允许除预安装应用（由设备或系统映像制造商预安装）以外的任何应用以写入方式访问外置 SD 卡。对于此问题，我们完全无力解决。唯一的解决办法是对您的设备执行 root 操作并更改权限，或者升级到 Android 5+ 并使用 SAF.
